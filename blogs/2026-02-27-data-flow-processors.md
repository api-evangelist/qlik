<p>In this blog post, I will review some data flow processors that can be used to prepare your data in a data flow. Let’s start by quickly reviewing what a data flow is. In Qlik Cloud Analytics, a data flow is a no-code experience that visually allows you to prepare your data with drag and drop capabilities. It is intuitive and easy to use and does not require the user to have scripting experience. Data flow processors, along with sources and targets, are used to build a data flow. Each processor handles a specific data transformation task. <a href="https://help.qlik.com/en-US/cloud-services/Subsystems/Hub/Content/Sense_Hub/DataFlow/List-of-data-flow-processors.htm" rel="noopener" target="_blank">Here</a> you will find a full list of the data flow processors available.</p>
<p>This blog will touch base on a few processors to familiarize you with how they work and how easy they are to use. To begin, a data flow must first be created. There is more than one way to do this. From the Qlik Cloud Analytics catalog, click on the + Create new button and select Data flow or navigate to Prepare data from the menu and click on the add Data flow button at the top of the page.</p>

<p><span>On the left, there are sources, processors and targets. The source is the data input, the processors are the data transformation types, and the targets are data outputs. Before we can look at the processors, we need to select our input data from the data catalog or a connection. Once that is in place, we can begin to explore the processor options. There are several data flow processors – too many to review in this blog but I will review three of the them - the Filter processor, the Join processor and the Unpivot processor.</span></p>
<p><strong><span>Filter Processor</span></strong></p>
<p><span>The <a href="https://help.qlik.com/en-US/cloud-services/Subsystems/Hub/Content/Sense_Hub/DataFlow/data-flow-processor-filter.htm" rel="noopener" target="_blank">filter processor</a> filters data based on a condition. A processor can be added to the data flow canvas by dragging and dropping the processor onto the canvas or by clicking on the menu in the data source and selecting Add processor.</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-center" style="width: 999px;"><img alt="filter.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187110i3BDEAB83392CD4D0/image-size/large?v=v2&amp;px=999" title="filter.png" /></span></span></p>
<p>&nbsp;</p>
<p>If you drag and drop the processor onto the canvas, you will need to connect the dots between the input and processor. If you add it from your data source menu, the dots will automatically be connected for you.</p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-center" style="width: 904px;"><img alt="filter prop.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187111i123565FB23B5850C/image-size/large?v=v2&amp;px=999" title="filter prop.png" /></span></span></p>
<p>&nbsp;</p>
<p><span>Each processor has a properties panel where the processor can be configured.&nbsp; In this example, let’s use the filter to select employees who live in the United States. To do this, first select the field to process – Country. There is an option to apply a function but one is not needed in this example.&nbsp;&nbsp;The operator will be equal, and the Value will be United States. Once the properties are entered, click the Apply button to save.</span></p>
<p><span><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 371px;"><img alt="filter complete.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187112i0D84D495A47C3870/image-size/large?v=v2&amp;px=999" title="filter complete.png" /></span></span></span></p>
<p><span><span>At the bottom of the page, I can preview the script (matching and not matching records) for the filter processor I just applied and see a preview of the data.</span></span></p>
<p><span><span><span><span class="lia-inline-image-display-wrapper lia-image-align-center" style="width: 999px;"><img alt="preview.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187113i365325142A7979E5/image-size/large?v=v2&amp;px=999" title="preview.png" /></span></span></span></span></p>
<p>&nbsp;</p>
<p><span><span><span>From the filter processor menu, there are a few options for my next step as seen below.</span></span></span></p>
<p><span><span><span><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 521px;"><img alt="add matching.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187114i60CE9B1B04AF0B9E/image-size/large?v=v2&amp;px=999" title="add matching.png" /></span></span></span></span></span></p>
<p><span>Add matching target will add a target to the data flow for the records that match the Country = United States filter. Add non-matching target will add a target to the data flow for the records that do not match the Country = United States filter. Matching and non-matching processors can also be added. For this example, I will add a matching target and in the properties panel, I will select the space, the extension (.qvd, .parquet, .txt or .csv) and the name of the target file. Like the sources, the target can be a data file or connection. Once I click Apply in the properties panel, I will see a message at the top right indicating that my flow is valid and ready to be run. Running the data flow will grab my Employees dataset, filter the data by country and store the results in a QVD named US Employess.</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 374px;"><img alt="emp target.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187115iC5BA3DFEA2957FC2/image-size/large?v=v2&amp;px=999" title="emp target.png" /></span></span></p>
<p><span>I now have a data file that has been transformed and prepared for use.</span></p>
<p><strong><span>Join Processor</span></strong></p>
<p><span>Now, let’s look at how we can join two data inputs into one data output. To do this, two data inputs are required. In the example below, ARSummary and ARSummary-1 are the two data inputs.</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-center" style="width: 933px;"><img alt="join flow.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187116iB1E87586B0BB9029/image-size/large?v=v2&amp;px=999" title="join flow.png" /></span></span></p>
<p>&nbsp;</p>
<p>In the properties panel of the join processor, the join type is selected and the fields that should be used to link the two tables are selected. You can learn more about joins <a href="https://help.qlik.com/en-US/cloud-services/Subsystems/Hub/Content/Sense_Hub/DataFlow/data-flow-processor-join.htm" rel="noopener" target="_blank">here</a>. Once the target is added, the data flow can be run, and the result will be a single table with the records from the ARSummary table and the associated records from the ARSummary-1 table.</p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 376px;"><img alt="join prop.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187117iC1B704A55B5391C2/image-size/large?v=v2&amp;px=999" title="join prop.png" /></span></span></p>
<p><span><strong>Unpivot Processor</strong></span></p>
<p><span>If you are familiar with scripting, the <a href="https://help.qlik.com/en-US/cloud-services/Subsystems/Hub/Content/Sense_Hub/DataFlow/data-flow-processor-unpivot.htm" rel="noopener" target="_blank">unpivot processor</a> is like a crosstable load. It allows you to rearrange a table so that column data becomes row data. It can transform a table like this:</span></p>

<p><span>To this:</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 557px;"><img alt="after.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187119i7585ADF5D238578A/image-size/large?v=v2&amp;px=999" title="after.png" /></span></span></p>
<p><span>Here is an example data flow with the unpivot processor:</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-center" style="width: 844px;"><img alt="unpivot flow.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187120i2415EB5C2ACAA0E9/image-size/large?v=v2&amp;px=999" title="unpivot flow.png" /></span></span></p>
<p>&nbsp;</p>
<p><span>In the properties panel of the unpivot processor, there are only a few settings to update. The first is the unpivot fields. Here is where the fields that we want to unpivot are selected. In this example, we want the year to be stored as row level data so we select them all.</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 373px;"><img alt="unpivot fields.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187121i27E23A86A52CAC92/image-size/large?v=v2&amp;px=999" title="unpivot fields.png" /></span></span></p>
<p><span>The Attribute field name is the name we want to give to the unpivoted fields – in this case Year. The Value field name is the name of the data that is associated with the fields we are unpivoting – in this examples Sales.</span></p>
<p><span><span class="lia-inline-image-display-wrapper lia-image-align-inline" style="width: 376px;"><img alt="unpivot prop.png" src="https://community.qlik.com/t5/image/serverpage/image-id/187122iC6037BC1FC94C5CA/image-size/large?v=v2&amp;px=999" title="unpivot prop.png" /></span></span></p>
<p><span>After applying the changes and running the data flow, we will have a table transformed based on our specifications without any code.</span></p>
<p><span>In this blog post, we touched upon three of the many processors that can be used in a data flow. Note that a data flow can have many sources, processors and targets – it all depends on your needs. The visual interface of a data flow makes it easy to prepare your data without any code in an appealing design that is easy to follow. Try it out!<br /></span></p>
